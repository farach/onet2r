## Decisions Made
- Keep live API calls opt-in for vignettes via `ONET2R_RUN_LIVE_VIGNETTES=true`, so local API keys do not make builds network-dependent.
- Publish executable representative examples by default, using small tibbles and `head()`-style bounded output to keep articles readable.
- Treat O*NET longitudinal comparisons cautiously: source-date changes, taxonomy splits/merges, and domain/method changes are surfaced as explicit flags.
- Use equal weights for the initial crosswalk bridge; employment-weighted bridging intentionally errors until an authoritative employment input is added.
- Keep package-source cleanup strict: generated pkgdown/vignette artifacts and local agent files stay ignored or out of the repository.
- Use Bootstrap 5/Flatly, Inter typography, and JetBrains Mono code styling for a cleaner pkgdown site.
